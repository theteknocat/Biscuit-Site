Destroy and restart the session, with the optional feature of backing up the session first and then restoring it afterwards.


Destroy and restart the session, with the optional feature of backing up the session first and then restoring it afterwards. The backup and restore functions are for when you want to clear a particular session variable and then restart the session for security reasons, but want to carry forward information from the old session to the new one.

		
Parameters

	
    
			
  • $backup $backup: bool Optional - Whether or not to backup the session first

  • $exceptions $backup_keepers: array Optional - An array of session variables you don't want to keep when backing up
