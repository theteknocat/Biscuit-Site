Method define_access_levels (line 459)

void define_access_levels( )

Map global variable names to access level numbers so that scripts and plugins do not need to rely on every site have specific level numbers.

Map global variable names to access level numbers so that scripts and plugins do not need to rely on every site have specific level numbers. Default globals that are expect by the system are PUBLIC_USER, WEBMASTER, ADMINISTRATOR and SYSTEM_LORD, so errors may occur if these four levels do not exist in the database. SYSTEM_LORD is the programmer level, and is intended for use when programmer-level functions are incorporated such as install/updating scripts.
